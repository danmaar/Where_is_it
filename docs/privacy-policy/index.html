1. About the App

Where is it? is a local inventory app designed to help users keep track of items and storage locations. The app does not require account creation, registration, or sign-in.

2. What Data the App May Process

The app may store information that the user chooses to add, including:

  • item names;
  • storage location names;
  • tags;
  • item quantities;
  • notes;
  • photos attached to items and/or locations;
  • favorite marks;
  • search history;
  • local app settings;
  • local backup files created through the app's export and import features.

The app does not require the user's name, phone number, email address, location data, or similar information for its core functionality. However, information entered by the user in notes, names, or photos may contain personal data if the user chooses to include it.

3. Camera and Photo Access

The app may request access to the camera only when the user explicitly chooses to take a photo in order to attach it to an item.

The app may request access to photos or the device gallery only when the user explicitly chooses to select an image to attach to an item.

The app does not use the camera, gallery, or photos in the background.

4. Where and How Data Is Stored

All app data is stored locally on the user's device.

The app does not transmit user data to the developer's servers and does not use cloud synchronization. The app does not sell or transfer personal data to third parties.

If the user uses the backup export feature, the backup file is created and stored locally in a location chosen by the user. Such files remain under the user's control.

5. Sharing with Third Parties

The app:

  • does not send user data to the developer's servers;
  • does not require registration or login;
  • does not sell data;
  • does not share personal data with third parties.

6. Data Retention and Deletion

Data is stored locally on the user's device until the user:

  • deletes specific records or photos;
  • clears the app's data;
  • uninstalls the app;
  • deletes locally created backup files.

7. Security

The developer takes reasonable steps to protect data within the app's architecture. Because data is stored locally on the user's device, data security also depends on the user's device, including device access controls, screen lock, encryption, and other security features used by the user.

No method of storage can guarantee absolute security. However, the app does not transmit data outside the device without the user's explicit action.

8. Intended Audience

The app is intended for a general audience and is not specifically directed to children.

9. Changes to This Policy

The developer may update this Privacy Policy from time to time. The current version of the policy should be published at the active link provided for the app.
